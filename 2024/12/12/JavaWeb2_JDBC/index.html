先搞个MySQL玩玩

banner: mysql 8.0.37

  1. 包治百病[https://blog.csdn.net/qq_43342301/article/details/91288891]
    1. 其中跳过密码登录再本地需要自己指定文件并administrator权限，mysqld --defaults-file="D:\MySQL_Server_8_Data\my.ini" --console --skip-grant-tables --shared-memory
    2. 其他照着时评里说的就ok
  2. 创建一个数据库，这里有个项目需要跑，刚刚好，注意大小写
    1. 字符集与校对规则collation
      1. 字符集定义了字符的编码方式，而校对规则决定了这些字符如何被比较和排序。
        1. 每个字符集至少有一个对应的校对规则，但通常会有多个，以满足不同语言和地区的需求。
        2. 当选择一个字符集时，默认会应用该字符集的默认校对规则，但你可以根据需要为数据库、表或列指定不同的校对规则。
      2. COLLATE后缀含义：
        1. _ci: case insensitive 不区分大小写
        2. _cs：case sensitive 区分大小写
        3. _ai： accent insensitive 不区分重音
        4. _as: accent sensitive 区分重音
        5. _bin：binary case sensitive collation，binary 以二进制方式，区分大小写和重音符号。
      3. MySQL常用排序规则有：utf8mb4_general_ci、utf8mb4_unicode_ci、utf8mb4_bin、utf8mb4_0900_ai_ci
        1. _ai_ci：按照特定语言或地区方式比较字符串，不区分大小写和重音符号。
        2. _unicode_ci： 按 Unicode 标准方式比较字符串，不区分大小写和重音符号。
        3. _general_ci：按一般方式比较字符串，不区分大小写和重音符号。
    2. 大小写敏感性
      1. 一篇不错的文章，详细看里面的就ok[https://blog.csdn.net/weixin_49114503/article/details/136523591]
      2. 数据库和表名/表别名
        1. 查看数据库名和表名的大小敏感性show variables like "%case%";
          1. 在Unix系统中，数据库名和表名的大小写是敏感的在
          2. Windows系统中，数据库名和表名的大小写不敏感，存储和查找时会转换为小写。
          3. 在macOS系统中，数据库名和表名的大小写是敏感的，但在查询时会转换为小写。
          4. 在大小写不敏感的系统中（Windows和macOS），不能将lower_case_table_names设置为0。
          5. 可以通过修改配置文件或设置字段属性来控制大小写敏感性。
        2. 修改库名及表名大小写敏感
          1. mysql8.0安装初始化之后是不能更改lower_case_table_names的(官方文档)：In MySQL 8.0, the lower_case_table_names variable can only be configured when the MySQL server is initialized
          2. 解决方法：
            1. 方法一：在初始化的时候就要设置好这个变量的值
            2. 方法二：删除数据目录，重新初始化(执行命令前切记先备份好数据，此方法并没有亲自试验过，切记慎重使用)
          3. 一般不用修改到数据库名与表名
      3. 列名大小写不敏感
        1. 列名与列的别名在所有的情况下均是忽略大小写的 ,不管是Linux系统还是Windows系统下MySQL的列名都默认不区分大小写
      4. 字段大小写敏感与排序
        1. 排序与校对规则：在比较时候使用的
        2. 字段内容大小写敏感
          1. 不管是windows还是Linux系统，mysql在任何系统上字符串内容默认是不区分大小写的。若想要字段内容区分大小写，可以设置字段编码大小写敏感
          2. 修改整个表的编码集
            1. 查看show table status from <database_name/schema> like '<table_name>';
            2. 修改alter table <table_name> character set <encoding_format> collate <encoding_rules>;编码集合可以是utf8_general_ci
          3. 修改某个字段的编码集
            1. 查看show full columns from table_name where field='field_name'
            2. 修改alter table <table_name> modify <field_name> <field_type> character set <encoding_rule> not null;比如alter table tb modify name varchar(10) character set utf8 not null;
          4. 在查询中指定是否敏感
            1. 敏感SELECT * FROM test1 WHERE name like '%B%' COLLATE utf8mb4_bin;
            2. 不敏感SELECT * FROM test1 WHERE name like '%B%' COLLATE utf8mb4_general_ci;

Transaction: 事务

Batch: 批处理

  1. Batch与Transaction的关系
    1. 批处理一般不支持多个事务，否则难以保证一致性，对资源锁的占有还有管理复杂度这些问题
    2. 多个/一个批处理构成一个事务

Spring中访问数据库

JdbcTemplate

  1. 对JDBC接口的进一步封装，旨在对资源的声明周期进行自管理

声明式事务

  1. 通过配置文件或注解来指定哪些方法应该运行在一个事务上下文中，而不是在业务逻辑代码中显式地编写事务控制语句（如beginTransaction()、commit()或rollback()）
  2. 事务的传播是在同一个线程中通过ThreadLocal实现的

DAO: Data Access Object

  1. 顾名思义：实际上就是与数据访问相关的类对象，提供curd操作，实现对数据访问的封装，将数据访问层与业务逻辑层相分离(术业有专攻)，有和实体对象进行了绑定，不再单纯的数据访问操作，但是又和实际上的model进行区别，总结就桥梁
  2. 嵌套结构与对应功能

ORM: Object Relational Mapping

概念知一二

  1. 核心思想: 将程序中的对象映射到关系型数据库中的表
  2. 实现
    1. 实体类与数据库表的映射: 类的属性与数据库表的列一一对应。每个对象实例对应数据库表中的一条记录
    2. CURD操作: 操作这些对象时，ORM框架会自动生成相应的SQL语句，并通过数据库连接执行这些语句，从而实现数据的插入、更新、删除和查询

Mybatis

  1. Mybatis是半自动化的ORM框架，但它并不像 Hibernate 那样完全自动化地将对象映射到数据库表中，更像是一个SQL映射框架

    1. SQL语句的自控制: 与全ORM框架不同的是，MyBatis并不自动生成SQL语句，允许开发人员直接书写 SQL 语句，并通过绑定参数和结果集映射来实现对象关系映射的功能

    2. 对象映射: 可以将查询结果映射成 Java 对象，但不会自动建立对象之间的关系
